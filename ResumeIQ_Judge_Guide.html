1. What Is ResumeIQ?

ResumeIQ is a browser-based AI resume screener that tells a candidate exactly why their resume is being filtered out by Applicant Tracking Systems (ATS) — and then fixes it automatically, in under 45 seconds, with zero account required and nothing stored server-side.

The Problem It Solves: 75% of resumes are rejected by ATS bots before a human ever sees them. Most job seekers have no idea which keywords are missing, which ATS rule they're tripping, or how their bullet points score. ResumeIQ makes the invisible visible — and then acts on it.

2. The Complete User Journey

1 Upload + Paste

The user lands on the homepage and does two things: uploads their PDF resume and pastes the job description they're targeting. There is no login, no sign-up, no email. It works instantly from day one.

2 PDF Extraction (Client-Side, Privacy-First)

The PDF never leaves the browser. The app loads PDF.js from CDN and extracts the raw text content entirely in the browser tab. This is why the "Zero Data Stored" badge is genuine — the file bytes are processed in memory and discarded.

Special engineering attention was paid to multi-column PDF layouts (the kind most job seekers download from Canva or Overleaf). The custom resume parser pre-processes merged column text by detecting 20 known ALL-CAPS section header phrases and re-inserting line breaks before them, so sections like CAREER OBJECTIVE, TECHNICAL SKILLS, and ACHIEVEMENTS are correctly separated even when PDF.js collapsed them into a single paragraph.

3 AI Analysis Engine

The extracted text and job description are sent to a Supabase Edge Function running on Deno, which calls Gemini 2.0 Flash via the Google AI API. The prompt instructs the model to return a strict 7-field JSON schema: 

matchScore         → 0–100 numeric score with keyword/skill/experience sub-breakdowns
resumeKeywords     → what the resume actually contains (technical, soft, tools, role)
jdKeywords         → what the JD demands
gaps               → missing keywords ranked by importance to this specific role
sectionFeedback    → strength + weakness + exact action for every resume section
atsSimulation      → which keywords ATS bots detect vs. silently miss + why
rewriteSuggestions → original bullets paired with improved ATS-optimised versions

Dual AI Redundancy: If Gemini hits a rate limit, the app automatically retries the identical call against Groq LLaMA-3.3-70b (OpenAI-compatible API) as a transparent fallback. Users never see a failure. If both are unavailable, a local heuristic analysis runs in-browser. The product never goes down.

5 AI Auto-Fix Engine (3-Step Pipeline)

Clicking "Auto-Fix My Resume with AI" launches a sequential 3-step AI engine that improves the entire resume programmatically:

  • Step 1 — Keyword Injection: Takes every keyword from the Gap Analysis and adds it to the most appropriate existing Skills category. Deduplicates against what's already present. Creates a "Key Skills" catch-all category if required.
  • Step 2 — Rewrite Substitution: Fuzzy-matches every existing resume bullet against the AI suggestions using a 60%-word-overlap algorithm. Replaces matching bullets with the stronger, ATS-optimised versions from the analysis.
  • Step 3 — AI Action-Verb Polish: Sends every remaining bullet to Gemini as a single JSON batch request. Restructures each bullet to open with a strong action verb and include measurable outcomes, aligned to the specific JD. Returns the improved set in full.

The widget displays live animated progress across all three steps: Injecting keywords… → Applying rewrites… → AI polish… → Done!

6 Resume Editor + Live Preview + PDF Export

After analysis or Auto-Fix, the user can open the full Resume Editor with one click:

  • Split-panel layout: 400px tabbed editor sidebar + full-width live-updating PDF preview that reflects every keystroke instantly
  • ✨ AI Smart Fill: Sends raw resume text to Gemini with a schema-exact JSON extraction prompt. Maps all 40+ fields (name, email, phone, location, LinkedIn, GitHub, summary, experience, education with GPA, grouped skill categories, projects with tech stacks, custom sections) in one 5-second click
  • Per-bullet AI Rewrite: Every bullet has a wand button that rewrites it against the JD on the spot
  • Style Panel: 3 templates, 3 fonts, 8 accent colour presets + custom hex, font size, spacing options
  • PDF Export: Captures the live DOM, wraps it in print-ready HTML with @page margins, and opens a ready-to-print popup — no server, no library, no cost

The 3 Resume Templates are production quality:

  • Classic — Jake's Resume-style single column, fully ATS-safe
  • Modern — Coloured sidebar with contrasting main content area
  • Minimal — Editorial style with thin dividers and generous whitespace

4. Why This Project Deserves to Win

1. Solves a Real, Quantified Problem

ATS rejection is the #1 barrier in modern job hunting. 75% of resumes never reach a human reviewer. Millions of qualified candidates are filtered out yearly — not because they're underqualified, but because their resume uses different phrasing than what the ATS was tuned to. ResumeIQ makes the ATS's black box completely transparent.

2. End-to-End — Not Just a Chatbot Wrapper

Most "AI resume tools" are just ChatGPT with a text box. ResumeIQ is a complete 12-step pipeline:

Upload PDF → Extract Text → Analyse → Score → Compare Keywords → Simulate ATS → Identify Gaps → Per-section Coaching → Rewrite Bullets → Auto-Fix (3 steps) → Edit in live editor → Export PDF

That is 12 distinct steps, fully automated, in one seamless product, in under 45 seconds.

3. Dual AI Redundancy = Production Reliability

The automatic Gemini → Groq fallback means the product never fails due to quota limits. This is production-level reliability engineering. No competitor at hackathon level has a live AI fallback chain with a local heuristic as a final safety net.

4. Privacy by Architecture, Not Policy

No user data is ever stored — not even transiently. The PDF is processed in the browser. The text is sent to a stateless, ephemeral edge function. The response is discarded after rendering. Zero database writes. This isn't a privacy policy — it's privacy enforced through architecture.

5. Three Distinct Layers of AI Intelligence

  • Layer 1: Supabase Edge Function + Gemini — structured 7-field JSON analysis with JD-specific grounding and specificity rules
  • Layer 2: Auto-Fix 3-step pipeline — keyword injection + fuzzy bullet matching (60% threshold) + AI action-verb restructure batch call
  • Layer 3: AI Smart Fill in Editor — schema-driven extraction mapping raw resume text to 40+ structured fields in one call

6. Production-Quality UX Engineering

  • Animated score gauge with spring physics via CSS cubic-bezier transitions
  • Per-field focus persistence in editor — correct React hook architecture, all state hoisted to avoid remount focus loss
  • Live resume preview updates on every keystroke with zero visible lag
  • Multi-column PDF parsing with pre-processing heuristics for real-world resume layouts
  • Responsive, accessible dark-first design with vibrant electric blue / cyan / fuchsia palette

7. Zero Dependency for Core Value

PDF export, resume editing, scoring — none require a paid external service beyond the AI API calls. The product keeps working even if Supabase is unavailable (local fallback). Every feature delivers value without vendor lock-in.

8. Engineering Discipline Under Pressure

The product was built iteratively with rapid feedback loops. Every discovered bug was diagnosed, fixed, and prevented — multi-column PDF parsing, React hook ordering violations, input focus loss, AI rate limit handling, color-system overhaul. This demonstrates not just technical skill but engineering discipline.

6. Live Demo Script (3 Minutes)

Suggested flow for judges evaluating the live product:

  1. Landing Page — Observe the animated hero, mesh gradient background, and feature grid. Note the "No account · 100% private · Gemini + Groq" navbar badge.
  2. Upload + Paste — Drop any text-based PDF resume onto the upload zone. Paste a software engineer job description into the right panel. Click "Try sample" to auto-fill a demo JD if preferred.
  3. Analyse — Click "Analyse My Resume". Watch the animated step indicators: Extracting PDF → Reading JD → Matching skills → Generating feedback. Results arrive in 15–30 seconds.
  4. Results Scroll — Scroll through: animated gauge → keyword side-by-side → ATS simulation → section feedback → rewrite panel. Each card is specific to the uploaded resume and JD, never generic.
  5. Auto-Fix — Scroll to "AI Resume Optimizer". Click "Auto-Fix My Resume with AI". Watch the 3-step live progress. Observe the score projection update.
  6. Editor — Click "Preview & Download Your Optimised Resume". The editor opens pre-filled with the AI-improved data. Click "✨ AI Smart Fill" to watch all 40+ fields populate in ~5 seconds.
  7. Style — Switch templates (Classic → Modern → Minimal), change accent colour, adjust font. Observe the live preview update instantly with every change.
  8. Export — Click "Download PDF". A print-ready resume opens in a popup, ready for Ctrl+P / Cmd+P.
Key talking points during demo: Point out that the ATS Simulation card explains why each keyword was missed (not just which ones). Point out the Auto-Fix progress shows each of the 3 AI steps live. Point out that closing the browser tab discards all data permanently.
